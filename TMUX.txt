Tmux Cheat Sheet

  Essentials (memorize these first)

  ┌──────────────────┬──────────────────────────────────────────────┐
  │     Shortcut     │                 Description                  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+D           │ Exit shell — closes pane (or window/session  │
  │                  │ if it was the last pane)                     │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ exit             │ Same as Ctrl+D                               │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B d         │ Detach — leave session running in background │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B ?         │ Show all key bindings (q to quit)            │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B :         │ Open tmux command prompt                     │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B x         │ Kill current pane (asks confirmation)        │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B &         │ Kill current window (asks confirmation)      │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ tmux kill-session -t name │ Kill a session by name              │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ tmux kill-server │ Kill ALL sessions and the tmux server        │
  └──────────────────┴──────────────────────────────────────────────┘

  Tip: scroll with the trackpad/wheel to auto-enter copy mode
  (q or Esc to exit). mouse on is set in ~/.tmux.conf.

  Multiple Servers

  Each server is a separate tmux process with isolated sessions.

  ┌─────────────────────────────────┬──────────────────────────────────┐
  │            Command              │            Description           │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work                    │ Create/attach to 'work' server   │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work new -s main        │ New session on 'work' server     │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work a                  │ Attach to 'work' server          │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work a -t mysession     │ Attach to specific session       │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work a -d               │ Attach and detach others         │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ tmux -L work kill-server        │ Kill entire 'work' server        │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │ pgrep -a tmux                   │ List running tmux servers        │
  └─────────────────────────────────┴──────────────────────────────────┘

  Note: 'tmux a' is shorthand for 'tmux attach'. Without -L, connects
  to the default server. Use -L <name> for named servers.

  Text/Copy

  Two flows are configured (~/.tmux.conf):

  A) Mouse drag — drag with the trackpad/wheel to select; release
     auto-copies to the macOS clipboard and exits copy mode.
     Paste with cmd+V anywhere outside tmux, or Ctrl+B ] inside tmux.

  B) Vi-style copy mode (mode-keys vi):

  ┌──────────────────┬──────────────────────────────────────────────┐
  │     Shortcut     │                 Description                  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B [         │ Enter copy mode                              │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ h j k l / arrows │ Move cursor                                  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ w / b            │ Next / previous word                         │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ 0 / $            │ Start / end of line                          │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ g / G            │ Top / bottom of buffer                       │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ /  ?             │ Search forward / backward (n / N to repeat)  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ v                │ Begin selection                              │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ V                │ Line-wise selection                          │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+v           │ Block (rectangle) selection                  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ y  or  Enter     │ Yank selection → system clipboard, exit      │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ q  or  Esc       │ Exit copy mode without copying               │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ Ctrl+B ]         │ Paste tmux buffer into the current pane      │
  └──────────────────┴──────────────────────────────────────────────┘

  Notes:
   - set-clipboard on is enabled, so OSC 52-aware terminals (Ghostty)
     get the selection even without pbcopy running.
   - Holding Option (⌥) while dragging bypasses tmux mouse capture
     and gives you the terminal's native selection instead — handy
     when you want to copy across multiple panes at once.

  Multiple Servers (separate socket = separate server)

  Each tmux server is identified by its socket. By default everything
  shares one socket, so all sessions live in one server. Use -L (named
  socket) or -S (explicit socket path) to start an independent server
  with its own sessions, options, plugins state, and copy buffers.

  ┌─────────────────────────────────────┬──────────────────────────────┐
  │              Command                │          Description         │
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -L work new -s main            │ Start server on socket 'work'│
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -L work ls                     │ List sessions on 'work'      │
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -L work attach -t main         │ Attach to 'work' server      │
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -S /tmp/tmux-other new         │ Server at explicit path      │
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -L work kill-server            │ Kill just the 'work' server  │
  ├─────────────────────────────────────┼──────────────────────────────┤
  │ tmux -L test -f ~/test.tmux.conf    │ Sandbox a different config   │
  └─────────────────────────────────────┴──────────────────────────────┘

  Notes:
   - prefix s only lists sessions in the current server.
   - Named sockets live in $TMUX_TMPDIR (or /tmp/tmux-<uid>/).
   - Useful for isolating work/personal contexts or testing a config
     without touching your main session tree.
